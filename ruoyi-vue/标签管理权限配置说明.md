# 标签管理权限配置说明

## 概述
本文档详细说明了如何完善博客系统中标签管理的路由和权限配置。

## 当前配置状态

### ✅ 已完成配置

1. **后端权限控制** (`BlogTagController.java`)
   - `system:tag:list` - 标签列表查询权限
   - `system:tag:query` - 标签详情查询权限
   - `system:tag:add` - 标签新增权限
   - `system:tag:edit` - 标签修改权限
   - `system:tag:remove` - 标签删除权限

2. **前端权限指令** (`/views/system/tag/index.vue`)
   - 新增按钮：`v-hasPermi="['system:tag:add']"`
   - 修改按钮：`v-hasPermi="['system:tag:edit']"`
   - 删除按钮：`v-hasPermi="['system:tag:remove']"`

3. **路由配置** (`/router/index.js`)
   - 标签管理路由已配置权限：`permissions: ['system:tag:list']`
   - 博客管理模块所有路由已统一配置权限

```bash

use ry-vue;

# 执行菜单权限配置脚本
source /home/nevell/code/newblog/ruoyi-vue/sql/blog_tag_menu.sql;
```

### SQL脚本内容说明

脚本将创建以下菜单权限：

1. **博客管理主菜单** (menu_id: 2000)
   - 菜单名称：博客管理
   - 父菜单：0（根菜单）
   - 排序：10

2. **标签管理菜单** (menu_id: 2003)
   - 菜单名称：标签管理
   - 父菜单：2000（博客管理）
   - 权限标识：`system:tag:list`
   - 组件路径：`system/tag/index`

3. **按钮权限**
   - 标签查询 (20030)：`system:tag:query`
   - 标签新增 (20031)：`system:tag:add`
   - 标签修改 (20032)：`system:tag:edit`
   - 标签删除 (20033)：`system:tag:remove`
   - 标签导出 (20034)：`system:tag:export`

4. **角色权限分配**
   - 为管理员角色（role_id: 1）分配所有标签管理权限

## 权限验证流程

### 1. 用户登录
- 系统获取用户角色和权限列表
- 权限信息存储在用户状态中

### 2. 路由权限过滤
- 前端路由系统根据用户权限过滤可访问的路由
- 只有拥有 `system:tag:list` 权限的用户才能看到标签管理菜单

### 3. 按钮权限控制
- 使用 `v-hasPermi` 指令控制按钮显示
- 无权限的用户看不到对应操作按钮

### 4. 后端接口权限验证
- Spring Security 通过 `@PreAuthorize` 注解验证接口权限
- 无权限的请求返回 403 错误

## 常见问题排查

### 问题1：菜单不显示
**原因**：菜单权限未正确分配或路由配置错误
**解决**：检查数据库菜单配置和角色权限分配

### 问题2：按钮不显示
**原因**：前端权限指令配置错误或用户无对应权限
**解决**：检查 `v-hasPermi` 指令和用户权限列表

### 问题3：接口返回403
**原因**：后端权限注解配置错误
**解决**：检查 `@PreAuthorize` 注解配置
