# 博客架构重新规划

## 概述

本次重新规划旨在实现博客系统的路由统一、前端统一和菜单统一，提升系统的可维护性和扩展性。

## 架构规划

### 1. 路由统一

#### 前台博客路由 (`/src/router/blog.js`)
- 路径前缀：无（前台直接访问）
- 包含：首页、文章详情、分类文章、标签文章、归档、简洁版、关于页面
- 特点：公开访问，无需登录

#### 后台管理路由 (`/src/router/admin.js`)
- 路径前缀：`/admin`
- 包含：仪表盘、系统管理、系统监控、系统工具、博客管理
- 特点：需要登录和权限验证

#### 公共路由 (`/src/router/index.js`)
- 包含：登录、注册、错误页面、重定向等
- 特点：所有用户均可访问

### 2. 前端统一

#### 目录结构重组
```
src/views/
├── blog/                 # 前台博客页面
│   ├── index.vue        # 博客首页
│   ├── about.vue        # 关于页面
│   ├── simple.vue       # 简洁版博客
│   ├── archive/         # 归档页面
│   └── article/         # 文章相关页面
├── admin/               # 后台管理页面
│   ├── dashboard/       # 仪表盘
│   ├── system/          # 系统管理
│   ├── monitor/         # 系统监控
│   ├── tool/           # 系统工具
│   └── blog/           # 博客管理
└── common/             # 公共页面
    ├── login.vue       # 登录页面
    ├── register.vue    # 注册页面
    └── error/          # 错误页面
```

#### 配置统一
- **菜单配置** (`/src/config/menu.js`): 统一管理前后台菜单
- **API配置** (`/src/config/api.js`): 统一管理接口地址
- **路由配置**: 统一管理路由结构和权限

### 3. 菜单统一

#### 前台菜单
- 首页、分类、标签、归档、关于
- 基于用户角色动态过滤
- 支持权限控制

#### 后台菜单
- 按功能模块分组：博客管理、系统管理、系统监控、系统工具
- 基于RBAC权限控制
- 支持面包屑导航

## 优势

### 1. 路由统一
- **清晰的路径规划**: 前台使用根路径，后台使用`/admin`前缀
- **权限控制统一**: 统一的权限验证机制
- **SEO友好**: 前台路径简洁，利于搜索引擎优化

### 2. 前端统一
- **模块化组织**: 按功能模块组织文件结构
- **易于维护**: 前后台代码分离，便于团队协作
- **可扩展性强**: 新增功能只需在对应模块添加

### 3. 菜单统一
- **配置化管理**: 菜单配置集中管理，便于修改
- **权限集成**: 与RBAC权限系统深度集成
- **用户体验一致**: 前后台菜单风格统一

## 部署说明

### 开发环境
```bash
# 启动前端开发服务器
cd ruoyi-ui
npm run dev

# 启动后端服务
cd ruoyi-admin
mvn spring-boot:run
```

### 生产环境
```bash
# 构建前端
cd ruoyi-ui
npm run build

# 部署后端
cd ruoyi-admin
mvn clean package
java -jar target/ruoyi-admin.jar
```
