# 组件命名规范与路径规范

## 1. 概述

本文档定义了RuoYi-Vue项目中组件的命名规范和路径规范，旨在统一开发标准，简化组件加载逻辑，提高开发效率和代码可维护性。

## 2. 组件路径结构

### 2.1 总体目录结构

```
src/
├── views/
│   ├── admin/             # 后台管理组件
│   │   ├── dashboard/      # 仪表盘
│   │   ├── system/         # 系统管理模块
│   │   │   ├── user/       # 用户管理
│   │   │   ├── role/       # 角色管理
│   │   │   └── ...         # 其他系统管理子模块
│   │   ├── monitor/        # 监控模块
│   │   ├── tool/           # 工具模块
│   │   ├── blog/           # 博客管理
│   │   └── statistics/     # 统计模块
│   ├── blog/              # 前台博客组件
│   ├── error/             # 错误页面
│   └── ...                # 其他视图组件
```

### 2.2 组件路径规范

1. **后台管理组件路径**：`@/views/admin/{module}/{submodule}/index.vue`
   - 示例：`@/views/admin/system/user/index.vue`

2. **前台博客组件路径**：`@/views/blog/{component}/index.vue`
   - 示例：`@/views/blog/article/index.vue`

3. **特殊页面组件路径**：`@/views/error/{errorCode}.vue`
   - 示例：`@/views/error/404.vue`

## 3. 组件命名规范

### 3.1 组件文件命名

1. **主组件文件**：使用 `index.vue` 作为模块的主组件文件
2. **功能组件文件**：使用描述性功能的名称，遵循 `camelCase` 命名法
   - 示例：`userProfile.vue`, `roleAssignment.vue`

### 3.2 路由组件名称规范

路由配置中的组件路径应遵循以下格式：

1. **完整路径模式**：`admin/{module}/{submodule}/index`
   - 示例：`admin/system/user/index`

2. **简化路径模式**：当模块直接对应一个组件时，可使用 `admin/{module}/index`
   - 示例：`admin/dashboard/index`

## 4. 组件导入规范

### 4.1 loadView 函数使用规范

1. **标准路径格式**：
   - 后台组件：`admin/{module}/{submodule}`
   - 前台组件：`blog/{component}`

2. **特殊页面**：使用完整文件名，如 `404`, `401`

### 4.2 loadView 函数工作原理（性能优化）

**2023-11-16 性能优化说明：**

- **路径尝试优化**：优化后的函数最多只尝试2个路径，而非之前的10个路径
- **路径规范化**：移除不必要的后缀，统一处理格式
- **性能提升**：通过减少路径尝试次数，显著提高组件加载性能
- **错误处理**：提供更详细的错误日志，便于调试

**加载逻辑流程图：**

1. 特殊页面处理（404、401等）→ 直接返回已知路径
2. 路径规范化 → 移除不必要的后缀
3. 路径类型判断 →
   - 包含 `admin/` 前缀 → 提取实际路径并尝试标准后台路径格式
   - 包含 `blog/` 前缀 → 提取实际路径并尝试前台路径格式
   - 其他情况 → 默认作为后台管理组件处理
4. 最多尝试2个路径 → 主要路径 + 备选路径
5. 失败处理 → 返回404页面并记录错误日志

## 5. 路由配置规范

在路由配置中，`component` 属性应遵循以下格式：

```javascript
// 正确格式
{
  path: '/admin/system/user',
  component: 'admin/system/user',
  name: 'User',
  meta: { title: '用户管理', icon: 'user' }
}

// 禁止使用的格式
{
  path: '/admin/system/user',
  component: 'system/user',       // 缺少 admin/ 前缀
  name: 'User',
  meta: { title: '用户管理', icon: 'user' }
}

{
  path: '/admin/system/user',
  component: 'admin/system/user/index', // 多余的 index 后缀
  name: 'User',
  meta: { title: '用户管理', icon: 'user' }
}
```
