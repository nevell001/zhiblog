墨 Blog — 项目概述

基于 Vue 3 + Spring Boot + TypeScript + Element Plus 构建的现代博客系统。以阅读体验为核心，兼顾写作效率与社区互动，打造简洁、美观、高性能的内容平台。

设计理念

📖
阅读优先
文章正文使用衬线字体、1.9 行高、最大宽度 680px，确保长时间阅读舒适度
✦
简洁现代
大量留白、扁平设计、克制用色，视觉重点留给内容本身
⇆
响应式适配
桌面 / 平板 / 移动端三端适配，关键内容优先展示
⚡
性能优先
路由懒加载、组件按需引入、骨架屏、虚拟滚动保障流畅体验
◆
交互一致
统一使用 Element Plus 组件，表单、弹窗、分页等行为一致
◑
类型安全
全链路 TypeScript，API 响应、Props、Store 状态均有类型约束

组件封装原则
业务组件基于 Element Plus 二次封装，通过 TypeScript 接口约束 Props 类型。组件内处理样式定制和交互逻辑，对外暴露简洁的 API。组件文档通过 Vitepress 自动生成。

响应式策略

采用移动优先的响应式设计，三个断点覆盖桌面、平板、移动端。核心原则：内容优先级不变，布局自适应。

断点定义

🖥️
桌面端 lg
≥ 1024px
三栏 / 双栏布局完整展开，侧边栏可见，编辑器分屏
📋
平板端 md
768px - 1023px
隐藏次要侧栏，双栏变单栏 + 抽屉，编辑器单栏
📱
移动端 sm
< 768px
单栏布局，导航收为汉堡菜单，目录隐藏

各页面响应式适配方案

页面桌面 (≥1024px)平板 (768-1023px)移动 (<768px)
首页双栏：文章列表 + 侧边栏单栏：侧边栏移至底部单栏：汉堡菜单 + 列表流
文章详情三栏：目录 + 正文 + 作者卡双栏：正文 + 作者卡单栏：仅正文，目录隐藏
文章编辑分屏：编辑 + 预览单栏：编辑或预览切换单栏：仅编辑，预览弹窗
个人中心表格全列展开表格隐藏次要列卡片列表替代表格
登录注册左右分屏：品牌 + 表单左右分屏仅表单，品牌区隐藏
移动端适配细节
移动端导航使用 Element Plus ElDrawer 实现侧滑菜单；文章卡片缩略图在小屏下隐藏或缩小；评论区嵌套层级最多 2 层；表格在小屏下转为卡片列表展示。所有交互区域最小 44×44px，符合触控规范。

性能优化策略

关注首屏加载性能与页面切换流畅度，从代码分割、资源优化、渲染策略三个维度系统优化。

加载性能

策略方案预期收益
路由懒加载() => import('@/views/ArticleView.vue')首屏 JS 体积减少 60%+
组件按需引入Element Plus 自动导入 (unplugin-auto-import)组件 JS 减少 70%
骨架屏ElSkeleton 占位，内容加载后替换消除白屏感，FCP 提前
图片懒加载Intersection Observer + 渐进式加载首屏图片请求减少 80%
代码压缩Vite build + terser，Gzip/Brotli 压缩传输体积减少 70%
CDN 加速静态资源 CDN 分发，接口就近回源资源加载延迟 <100ms

渲染性能

策略方案适用场景
虚拟滚动ElVirtualList / vue-virtual-scroller文章列表、评论列表、通知列表
防抖节流搜索输入 300ms 防抖，滚动监听 16ms 节流搜索框、目录高亮
缓存策略Pinia + keep-alive 缓存已访问页面首页、文章详情返回不重新请求

⚡

🎯
CLS < 0.1
布局偏移控制，确保视觉稳定
草稿自动保存策略
双重保存：每 5s 同步到 localStorage（防意外关闭），每 30s 调用 API 持久化。保存状态实时显示（"保存中..." / "已保存 · 刚刚"）。页面离开前 beforeunload 提示。使用 AbortController 取消未完成的保存请求。
字体加载优化
Inter 和 Noto Serif SC 通过 Google Fonts 加载，使用 font-display: swap 避免 FOIT。首屏仅需 Regular 字重，Bold/Medium 字重按需加载。衬线字体仅用于文章正文区域，减少首屏字体请求。
