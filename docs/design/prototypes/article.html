Vue 3.4 新特性详解：defineModel 与 Props 解构稳定性

defineModel 稳定化

Vue 3.4 中，defineModel 宏正式从实验状态转为稳定 API。它简化了父子组件间的双向绑定，不再需要手写 props + emit 的样板代码。

之前我们需要这样写：

复制const props = defineProps<{ modelValue: string }>()
const emit = defineEmits<{ 'update:modelValue': [string] }>()

现在只需一行：

复制const model = defineModel<string>()
defineModel 不仅减少了代码量，更重要的是让组件的"受控/非受控"模式更加直观。

Props 解构响应性

这是一个期待已久的修复。在 3.4 之前，解构 props 会丢失响应性，需要借助 toRefs。现在 Vue 编译器会自动处理解构后的响应性保持。

解构后直接使用，无需 toRefs：

复制const { count, name } = defineProps<{
  count: number
  name: string
}>()

当父组件更新 count 或 name 时，子组件中解构出的变量会自动保持响应性。这得益于编译器对解构语法的转换——它会在编译时将解构访问替换为属性访问。

模板解析器重写

3.4 重写了模板解析器，解析速度提升约 2 倍。这对大型项目的编译性能有明显改善。

AST 优化

新的解析器生成更精简的 AST 节点，减少了不必要的包装层，使得后续的转换和代码生成步骤也更快。AST 节点数量减少了约 15%，遍历效率显著提升。

性能对比

在一个包含 1000 个节点的模板上测试，解析时间从 45ms 降至 22ms，编译总时间减少约 30%。这对使用 Vite 的开发体验尤为明显——HMR 速度直接受益于解析器提速。

  • 解析速度：45ms → 22ms（提升 2x）
  • 编译总时间：减少约 30%
  • AST 节点数量：减少约 15%
  • HMR 速度：提升约 25%

总结与展望

Vue 3.4 是一个以稳定性和性能为核心的版本。defineModel 和 Props 解构响应性的稳定化，让日常开发体验更加顺畅。模板解析器的重写则为未来的优化奠定了基础。

下一个大版本（3.5）预计将进一步优化响应式系统的性能，并可能引入 Vapor Mode——一种不依赖虚拟 DOM 的编译模式。让我们拭目以待。
