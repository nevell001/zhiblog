# 系统架构设计

## 一、整体架构

本博客系统采用前后端分离架构，后端基于 RuoYi-Vue（Spring Boot + MyBatis + Spring Security），前端采用 Vue.js + Element UI，数据库使用 MySQL，缓存采用 Redis，支持本地和云存储，部署支持 Docker。

```
[用户] <-> [前端 Vue.js] <-> [后端 RuoYi-Vue] <-> [MySQL/Redis/OSS]
```

- 前端与后端通过 RESTful API 通信，前端页面响应式设计，适配PC和移动端。
- 后端采用分层架构，包含控制层、服务层、数据访问层、通用工具层。
- 支持多用户、权限控制、内容管理、评论互动、日志监控等。

## 二、主要模块说明

- **用户与权限管理**：基于 Spring Security，支持多角色、权限分配、第三方登录扩展。
- **内容管理**：文章、分类、标签、归档、搜索，支持富文本/Markdown。
- **评论互动**：多级评论、点赞、审核、敏感词过滤。
- **站点管理**：站点信息、友情链接、公告、广告位。
- **媒体资源管理**：图片、附件上传与管理，支持本地/OSS。
- **日志与监控**：操作日志、登录日志、系统监控。
- **前台展示**：首页、文章页、分类页、标签页、归档页、个人中心、搜索。
- **扩展功能**：RSS、API、多语言、夜间模式。

## 三、技术选型

- 后端：Spring Boot、MyBatis、Spring Security、RuoYi 框架
- 前端：Vue.js、Element UI、Axios
- 数据库：MySQL
- 缓存：Redis
- 存储：本地/OSS
- 部署：Docker

## 四、数据流与交互

1. 用户通过前端页面发起请求（如登录、发文、评论等）
2. 前端通过 Axios 调用后端 RESTful API
3. 后端进行权限校验、业务处理、数据存储/查询
4. 结果返回前端，前端渲染页面

## 五、扩展性设计

- 所有业务模块均采用松耦合设计，便于后续功能扩展
- 支持插件/主题机制，便于个性化定制
- API 设计规范，便于对接移动端/第三方系统
